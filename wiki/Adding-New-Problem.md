# Adding a New Problem

This page describes the process for adding a new programming problem to the **Programming Solutions Universe**.

---

## 1. Before You Start

1. Check the [Problem Index](/questions.md) to see if a similar problem already exists.
2. If you’re unsure whether the new problem fits, open an [issue](https://github.com/DibyajyotiBiswal57/programs/issues) to discuss it.
3. Read the [Contributing Guide](/CONTRIBUTING.md) and [Code Style Guidelines](Code-Style.md).

---

## 2. Define the Problem

Each problem should include:

- A clear **title**
- A unique **problem ID [####_filename]**
- **Input** and **output** specifications
- Constraints and edge cases
- At least one example with sample input/output

Follow the format used by existing problems in [Problem Index](/questions.md)for consistency.

---

## 3. Add Problem Statement

Update the [Problem Index](/questions.md) to include your new problem.

---

## 4. Add Initial Solutions (Optional but Recommended)

To make the problem immediately useful:

1. Implement at least one solution in a supported language.
2. Place the solution in the correct language directory.
3. Ensure the solution follows [Code Style Guidelines](Code-Style.md).

---

## 5. Update Status

- Ensure problem metadata files are updated so the [Interactive Status Page](dibyajyotibiswal57.github.io/programs) reflects the new problem.

---

## 6. Submit Your Changes

1. Run any tests or checks described in the [Contributing Guide](/CONTRIBUTING.md).
2. Open a pull request describing:
   - The new problem
   - Any initial solutions you added
   - Any changes to metadata or scripts
3. Address feedback from reviewers.
