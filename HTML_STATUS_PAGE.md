# HTML Status Page Documentation

## Overview

This project includes an automated system for generating an interactive HTML status page that displays programming questions and their completion status across multiple languages.

## Files

### `generate_html.py`
Python script that generates `index.html` from `questions.md` and `status.md`.

**Usage:**
```bash
python3 generate_html.py
```

**What it does:**
1. Reads all questions from `questions.md`
2. Reads status badges from `status.md` (generated by `update_status.py`)
3. Generates a modern, responsive HTML page with:
   - Question cards in a 3-column grid
   - Collapsible status dropdowns
   - Smooth animations
   - Mobile-friendly design
   - Accessibility features

### `index.html`
The generated HTML status page. **Do not edit manually** - it will be overwritten.

To customize the page design, edit `generate_html.py` instead.

### `.github/workflows/update-html.yml`
GitHub Actions workflow that automatically updates `index.html` when:
- `questions.md` changes
- `status.md` changes
- `generate_html.py` changes
- Manually triggered from GitHub Actions UI

## Workflow

### Automatic Updates

1. When you push changes to `questions.md` or when `status.md` is updated, the workflow runs automatically
2. It executes `generate_html.py` to regenerate `index.html`
3. If `index.html` changed, it commits and pushes the update

### Manual Generation

To generate the HTML locally:

```bash
# First, ensure status.md is up to date
python3 update_status.py

# Then generate the HTML
python3 generate_html.py
```

## Design Features

### Visual Design
- **Card-based layout**: Each question in its own card
- **Color scheme**: Professional blue gradient with clean white cards
- **Typography**: System font stack for best performance
- **Animations**: 
  - Page fade-in on load
  - Cards slide up with stagger effect
  - Hover effects with lift and shadow
  - Smooth dropdown transitions

### Responsive Design
- **Desktop**: 3-column grid
- **Tablet**: 2-column grid (automatically adjusts)
- **Mobile**: Single column layout
- All elements scale appropriately

### Accessibility
- **ARIA labels**: Screen reader support
- **Keyboard navigation**: All interactive elements accessible via keyboard
- **Semantic HTML**: Proper heading hierarchy and structure
- **High contrast**: Readable text on all backgrounds

## Customization

### Changing Colors

Edit the CSS variables in `generate_html.py` (around line 85-95):

```css
:root {
    --primary-color: #2563eb;      /* Main blue */
    --secondary-color: #1e40af;    /* Darker blue */
    --accent-color: #3b82f6;       /* Accent blue */
    --background: #f8fafc;         /* Page background */
    --card-bg: #ffffff;            /* Card background */
    /* ... more variables ... */
}
```

### Changing Layout

Modify the grid settings in `generate_html.py` (around line 150):

```css
.questions-grid {
    grid-template-columns: repeat(auto-fill, minmax(350px, 1fr));
    gap: 25px;
}
```

### Adding Features

Edit `generate_html.py` and the `generate_html()` function to add new features or modify the structure.

## Troubleshooting

### HTML not updating automatically

1. Check if the workflow is enabled in GitHub Actions
2. Verify the workflow has permissions to commit (needs `contents: write`)
3. Check workflow logs for errors

### Badges not displaying

1. Ensure `status.md` is properly generated (run `python3 update_status.py`)
2. Check that badge URLs are valid Shields.io URLs
3. Verify network connectivity to img.shields.io

### Styling issues

1. Clear browser cache
2. Check browser console for CSS errors
3. Test in different browsers for compatibility

## Maintenance

### Adding a New Question

1. Add the question to `questions.md`
2. Run `python3 update_status.py` (or wait for automation)
3. Run `python3 generate_html.py` (or wait for automation)
4. Commit changes

### Updating Question Status

1. Add/modify program files in language folders
2. Run `python3 update_status.py` to update `status.md`
3. Run `python3 generate_html.py` to update `index.html`
4. Or just push changes and let GitHub Actions handle it

## Browser Support

The HTML page works on:
- Chrome/Edge (latest)
- Firefox (latest)
- Safari (latest)
- Mobile browsers (iOS Safari, Chrome Mobile)

Uses standard HTML5/CSS3/ES6+ JavaScript features. No external dependencies required.

## Performance

- **Page size**: ~440 KB for 90 questions
- **Load time**: < 1 second on broadband
- **Images**: Loaded from Shields.io CDN
- **JavaScript**: Minimal vanilla JS, no frameworks
